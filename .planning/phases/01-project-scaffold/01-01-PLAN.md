<task type="auto">
  <name>Task 1: Create backend directory, pyproject.toml, and virtual environment</name>
  <files>backend/pyproject.toml</files>
  <action>
1. Create the `backend/` directory at the project root (`/Users/Sheppardjm/Repos/dontCave/backend/`).

2. Create `backend/pyproject.toml` with this exact content:

```toml
[project]
name = "dontcave-backend"
version = "0.1.0"
description = "MP3-to-chord-chart personal tool - backend"
requires-python = ">=3.10"
dependencies = [
    "fastapi==0.135.1",
    "uvicorn==0.41.0",
]
```

Only include Phase 1 dependencies (fastapi, uvicorn). Future phases will add librosa, pychord, etc.

Do NOT use `[build-system]` — this is an app, not a library.

3. Create the virtual environment inside `backend/`:

```bash
cd backend
python -m venv .venv
```

4. Activate the venv and install dependencies:

```bash
source .venv/bin/activate
pip install --upgrade pip
pip install fastapi==0.135.1 uvicorn==0.41.0
```

**IMPORTANT:** Do NOT use `pip install "fastapi[standard]"` — it fails on Python 3.14 due to transitive binary dependency build failures. Install `fastapi` and `uvicorn` as separate packages.
  </action>
  <verify>
```bash

# Expected: 0.135.1

# Expected: 0.41.0 (or close)
```

  <done>backend/pyproject.toml exists with fastapi and uvicorn dependencies listed. backend/.venv exists with both packages importable.</done>

<task type="auto">
  <name>Task 2: Create FastAPI app with health endpoint and verify it runs</name>
  <files>backend/main.py</files>
  <action>
1. Create `backend/main.py` with this exact content:

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/health")
async def health():
    return {"status": "ok"}
```

That is the complete file. No imports beyond FastAPI. No CORS middleware — the Vite proxy handles CORS in dev (see Plan 02).

2. Start the server to verify it works:

```bash
cd /Users/Sheppardjm/Repos/dontCave/backend
source .venv/bin/activate
uvicorn main:app --reload --port 8000
```

3. In a separate verification step, confirm the health endpoint responds:

```bash
curl http://localhost:8000/health
# Expected: {"status":"ok"}
```

4. Stop the server after verification (Ctrl+C or kill the process).
  </action>
  <verify>
