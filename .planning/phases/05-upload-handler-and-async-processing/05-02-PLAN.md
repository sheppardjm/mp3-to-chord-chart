  Replace the entire contents of frontend/src/main.js with the upload form and submit handler.

  **HTML structure** (set via innerHTML on #app):
  ```
  <h1>dontCave</h1>
  <form id="upload-form">
    <label for="file-input">Select MP3 file:</label>
    <input type="file" id="file-input" accept=".mp3,audio/mpeg" required />
    <button type="submit" id="submit-btn">Analyze</button>
  </form>
  <p id="status"></p>
  <pre id="result"></pre>
  ```

  **JavaScript behavior:**
  1. Query the form, status element, result element, and submit button after innerHTML is set
  2. Add a submit event listener on the form with `async (e) => { e.preventDefault(); ... }`
  3. Inside the handler:
     a. Clear previous result: `resultEl.textContent = ''`
     b. Check fileInput.files.length, show "Please select an MP3 file." if empty, return
     c. Create FormData, append 'file' key with fileInput.files[0]
     d. Set loading state BEFORE await: `statusEl.textContent = 'Analyzing audio...'` and `submitBtn.disabled = true`
     e. try/catch/finally block:
        - `const response = await fetch('/api/analyze', { method: 'POST', body: formData })`
        - Do NOT set Content-Type header (browser must set it with multipart boundary)
        - If !response.ok: parse error body with `await response.json().catch(() => ({ detail: response.statusText }))`, set statusEl to `Error: ${err.detail || response.statusText}`, return
        - If response.ok: parse JSON, set statusEl to 'Analysis complete.', set resultEl to `JSON.stringify(data, null, 2)`
     f. catch block: `statusEl.textContent = 'Network error: ${err.message}'`
     g. finally block: `submitBtn.disabled = false`

  Keep `import './style.css'` as the first line.

  Do NOT set headers: {'Content-Type': 'multipart/form-data'} in the fetch call — this breaks the multipart boundary.
  Do NOT use XMLHttpRequest — use fetch + FormData.
  Do NOT add polling or WebSocket — a simple DOM text toggle satisfies the success criteria.
  </action>
