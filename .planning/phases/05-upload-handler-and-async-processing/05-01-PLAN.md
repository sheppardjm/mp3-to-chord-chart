<objective>
Add file validation (size and MIME type) and threadpool offloading to the /analyze endpoint so that oversized or non-MP3 uploads are rejected immediately, and the event loop is not blocked during pipeline processing.

Purpose: The existing /analyze endpoint accepts any file and blocks the event loop for 10-60 seconds. This plan adds the validation guards and moves the blocking pipeline into a threadpool so the server remains responsive.
Output: Updated backend/main.py with validation + run_in_threadpool wrapping.
</objective>

  <files>backend/main.py</files>
  <action>
  Modify backend/main.py to add three things:

  1. **Constants and imports** at the top of the file:
     - Add `from starlette.concurrency import run_in_threadpool` to imports
     - Add `MAX_FILE_SIZE = 50 * 1024 * 1024` (50 MB)
     - Add `ALLOWED_MIME_TYPES = {"audio/mpeg", "audio/mp3"}`

  2. **Validation guards** as the first two checks inside `analyze()`, BEFORE any file I/O:
     - Check `file.size`: `if file.size is not None and file.size > MAX_FILE_SIZE:` raise HTTPException(status_code=413, detail="File too large. Maximum allowed size is 50 MB.")
     - Check `file.content_type`: `if file.content_type not in ALLOWED_MIME_TYPES:` raise HTTPException(status_code=415, detail=f"Invalid file type '{file.content_type}'. Only MP3 files (audio/mpeg) are accepted.")
     - Order matters: size check first (cheap), then MIME check, then file I/O

  3. **Extract pipeline into `_run_pipeline()` and offload via threadpool:**
     - Create a module-level `def _run_pipeline(tmp_path: str) -> dict:` function containing the synchronous pipeline logic currently inline in `analyze()`: load_audio, detect_chords_pipeline, beat_track_grid, extract_beat_chroma, segment_song, build_sections, and the return dict
     - In `analyze()`, replace the inline pipeline with: `result = await run_in_threadpool(_run_pipeline, tmp_path)`
     - Keep temp file write (await file.read()) in the async endpoint (it uses await)
     - Keep the finally block for os.unlink(tmp_path) in the async endpoint (it owns the temp file lifecycle)
     - Initialize `tmp_path = None` before the try block and check `if tmp_path:` in finally (cleaner than `'tmp_path' in locals()`)
     - Re-raise HTTPException explicitly: add `except HTTPException: raise` before the generic `except Exception` to prevent validation errors from being wrapped in a 500

  Keep the /health endpoint unchanged. Keep all existing imports. The docstring should be updated to mention validation and async processing.

  Do NOT use asyncio.get_event_loop() or run_in_executor directly — use starlette.concurrency.run_in_threadpool.
  Do NOT use File(max_length=...) for size limiting — it does not work.
  Do NOT move temp file cleanup into _run_pipeline — cleanup stays in the endpoint's finally block.
